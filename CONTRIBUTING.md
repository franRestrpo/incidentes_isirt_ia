# Guía de Contribución

¡Gracias por tu interés en contribuir a **ISIRT-IA**! Queremos que contribuir sea lo más fácil y transparente posible.

## Cómo empezar

1.  **Fork del repositorio:** Crea tu propia copia del proyecto.
2.  **Configuración local:** Sigue la [Guía de Despliegue](docs/DESPLIEGUE_TROUBLESHOOTING.md) para configurar tu entorno.
3.  **Crea una rama:** Usa nombres descriptivos como `feature/mi-mejora` o `fix/error-especifico`.
4.  **Desarrolla:** Escribe código limpio, siguiendo las convenciones del proyecto (PEP 8 para Python).
5.  **Pruebas:** Asegúrate de que las pruebas existentes pasan y añade nuevas si es necesario (`pytest`).
6.  **Pull Request:** Envía tus cambios para revisión.

## Estándares de Código

-   **Python:** Usamos `flake8` para el linting. Asegúrate de que no haya advertencias.
-   **React/TS:** Usamos ESLint y Prettier.
-   **Commits:** Usa mensajes descriptivos en español o inglés.

## Reporte de Errores

Si encuentras un error, por favor abre un *Issue* describiendo:
-   Los pasos para reproducirlo.
-   El comportamiento esperado vs el actual.
-   Tu entorno (OS, versión de Docker, etc.).
